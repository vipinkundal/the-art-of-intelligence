Agents And Tools

An agent is an AI system that can plan steps, use tools, inspect results, and continue toward a goal.

The simple idea

A model alone produces text or predictions. An agent wraps that model with tools, state, instructions, and checks so it can take useful actions.

Plan The agent breaks the goal into steps and decides what information or action is needed next.
Use tools It calls search, files, APIs, code execution, or other capabilities outside the base model.
Check result It inspects tool output, updates state, retries when needed, and stops when the goal is satisfied.

Common tools

  • Search tools: Let the agent retrieve current or external information instead of relying only on the model's stored knowledge.
  • File tools: Let the agent inspect, create, and edit local artifacts while preserving a trace of what changed.
  • API tools: Let the agent read or update structured systems such as databases, calendars, issue trackers, or internal services.

Example

A coding agent can inspect files, edit code, run tests, and summarize the result instead of only explaining what a human should do.

Why it matters for AI

Tool use turns a model from a text generator into part of a working system. It introduces planning, action, feedback, and verification as core design concerns.

Learning target

You should be able to describe the difference between a model and an agent, name common tool types, and explain why feedback checks matter.
