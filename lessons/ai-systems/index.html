10. Agents And Tool-Using Systems

Agent systems combine models with tools, memory, workflows, recovery policies, permissions, and validation so generated decisions can become reliable actions.

Agent system loop

Decide The model plans, routes, or decomposes the next step from the current state.
Act safely Tools run behind schemas, permissions, sandboxes, retries, and budgets.
Verify Observations, logs, validators, and human approvals decide whether to continue.

10.1 Agent patterns

Study:

  • ReAct: Interleaves reasoning, tool actions, and observations so the agent can adapt after each result.
  • Planner-executor: Separates high-level planning from step execution, making long tasks easier to inspect and retry.
  • Router-specialist: Sends tasks to specialized prompts, models, tools, or workflows based on task type.
  • Generate-critique-revise: Produces an answer, critiques it, and revises it before returning the final output.
  • Generate-rank-verify: Creates multiple candidates, ranks them, and verifies the strongest option.
  • Reflection loops: Let an agent inspect past failures or results and update its next attempt.
  • State-machine agents: Move through explicit states, making control flow predictable and debuggable.
  • Workflow graphs: Represent agent steps as connected nodes with branching, retries, and dependencies.
  • Hierarchical agents: Use manager and worker roles so complex goals can be delegated to subtasks.
  • Multi-agent debate: Uses multiple agents to propose, challenge, and compare answers before selection.
  • Blackboard architectures: Let independent components read and write to a shared working memory.
  • Event-driven agents: React to messages, file changes, webhooks, or system events instead of only direct prompts.
  • Human-in-the-loop systems: Insert human review where stakes, ambiguity, or permissions require judgment.
  • Long-term and episodic memory: Store durable facts and past interactions so future decisions can use prior context.
  • Tool routing: Chooses the right tool or API for a step based on intent, input type, and available permissions.
  • Structured output: Forces model responses into schemas that code can validate and consume reliably.
  • Recovery and retry policies: Define what to do when tools fail, outputs are invalid, or progress stalls.

10.2 Production concerns for agents

Understand:

  • Idempotent tools: Tools should be safe to retry without duplicating side effects.
  • Permission boundaries: Limit what the agent can read, write, call, or change.
  • Sandboxing: Runs risky code or actions in restricted environments.
  • Tool schemas: Define exact inputs and outputs so model calls can be validated.
  • Timeouts: Stop slow or stuck tool calls before they block the workflow.
  • Retries and backoff: Repeat transient failures with spacing so systems can recover.
  • State persistence: Saves progress so long-running tasks can resume after interruption.
  • Durable execution: Ensures workflows survive process restarts, crashes, or delayed callbacks.
  • Context management: Controls what evidence, memory, and instructions fit into the model window.
  • Trace logging: Records decisions, tool calls, observations, and errors for debugging and audit.
  • Budget limits: Cap tokens, time, money, tool calls, or external side effects.
  • Human approval points: Require explicit review before sensitive actions are taken.
  • Prompt injection: Malicious instructions try to override the system through user-visible text.
  • Indirect prompt injection: Malicious instructions arrive through retrieved pages, files, emails, or tool outputs.
  • Exfiltration attacks: Attempts to make the agent reveal secrets or private data.
  • Confused-deputy problems: The agent misuses its authority on behalf of untrusted content.
  • Tool-result validation: Checks whether tool output is complete, expected, safe, and consistent.
  • Transaction management: Groups actions so partial completion does not leave systems inconsistent.
  • Rollback and compensation: Undo or counteract side effects when later steps fail.

For application integration, Model Context Protocol is a current open standard for connecting AI applications to tools, data sources, and external systems. It belongs in the engineering layer of the roadmap, not in the mathematical core of AI.

Learning target

You should be able to design agent workflows with explicit state, tool contracts, validation, human approval points, retry behavior, and security boundaries.
