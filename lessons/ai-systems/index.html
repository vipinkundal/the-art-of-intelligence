Phase 10

10. Agents And Tool-Using Systems

Agent systems combine models with tools, memory, workflows, recovery policies, permissions, and validation so generated decisions can become reliable actions.

10.1 Agent patterns

Study:

  • ReAct
  • Planner-executor
  • Router-specialist
  • Generate-critique-revise
  • Generate-rank-verify
  • Reflection loops
  • State-machine agents
  • Workflow graphs
  • Hierarchical agents
  • Multi-agent debate
  • Blackboard architectures
  • Event-driven agents
  • Human-in-the-loop systems
  • Long-term and episodic memory
  • Tool routing
  • Structured output
  • Recovery and retry policies

ReAct combines interleaved reasoning traces with actions and observations from an external environment.

Toolformer demonstrated self-supervised learning of when and how a language model can call external tools.

10.2 Production concerns for agents

Understand:

  • Idempotent tools
  • Permission boundaries
  • Sandboxing
  • Tool schemas
  • Timeouts
  • Retries and backoff
  • State persistence
  • Durable execution
  • Context management
  • Trace logging
  • Budget limits
  • Human approval points
  • Prompt injection
  • Indirect prompt injection
  • Exfiltration attacks
  • Confused-deputy problems
  • Tool-result validation
  • Transaction management
  • Rollback and compensation

For application integration, Model Context Protocol is a current open standard for connecting AI applications to tools, data sources, and external systems. It belongs in the engineering layer of the roadmap, not in the mathematical core of AI.

Learning target

You should be able to design agent workflows with explicit state, tool contracts, validation, human approval points, retry behavior, and security boundaries.
