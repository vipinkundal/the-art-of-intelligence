Phase 3

3. Classical Artificial Intelligence

Classical AI should not be skipped. It supplies many of the reasoning and planning mechanisms now being combined with language models and agents.

Classical AI problem-solving loop

Model the world States, rules, constraints, facts, goals.
Explore possibilities Search trees, game trees, proofs, plans.
Commit to action Choose a path, assignment, proof, or plan.

3.1 Search algorithms

Learn and implement:

  • Breadth-first search: Expands the shallowest states first, so it finds the shortest path when every step has the same cost.
  • Depth-first search: Follows one branch deeply before backtracking, which saves memory but can miss shorter answers without safeguards.
  • Depth-limited search: Caps how far DFS can go, making it useful when infinite branches or cycles are possible.
  • Iterative deepening: Repeats depth-limited search with larger limits, combining BFS-like completeness with DFS-like memory use.
  • Uniform-cost search: Expands the cheapest path so far, which is essential when actions have different costs.
  • Dijkstra's algorithm: Computes shortest paths from a start node when edge costs are non-negative.
  • Bidirectional search: Searches forward from the start and backward from the goal to reduce the explored state space.
  • Greedy best-first search: Chases the state that appears closest to the goal, making it fast but not necessarily optimal.
  • A*: Balances path cost so far with estimated remaining cost to find optimal paths when the heuristic is valid.
  • Weighted A*: Gives the heuristic extra influence, often trading perfect optimality for faster solutions.
  • Iterative Deepening A*: Uses increasing cost thresholds to get A* style guidance with lower memory usage.
  • Beam search: Keeps only the best few candidate states at each layer, useful when full search is too expensive.
  • Branch and bound: Prunes branches that cannot beat the best solution found so far.
  • Hill climbing: Moves to a better neighboring state repeatedly, which is simple but can get stuck on local optima.
  • Random-restart hill climbing: Runs hill climbing from many starting points to escape unlucky local optima.
  • Simulated annealing: Sometimes accepts worse moves early on, then gradually becomes more selective.
  • Tabu search: Remembers recent choices to avoid cycling back to the same unhelpful states.
  • Local beam search: Tracks several candidate states in parallel so promising regions can compete.

A* evaluation

f(n) = g(n) + h(n)

where g(n) is the path cost so far and h(n) estimates the remaining cost.

Understand

  • Admissible heuristics: Estimates that never overstate the remaining cost, preserving optimality for A*.
  • Consistent heuristics: Estimates that obey a triangle-inequality style rule, preventing A* from needing repeated repairs.
  • Completeness: Whether an algorithm is guaranteed to find a solution when one exists.
  • Optimality: Whether the solution found is guaranteed to be the best according to the objective.
  • Time and memory complexity: How search effort grows with branching factor, depth, and stored frontier size.
  • State-space representation: The choice of variables, actions, and transitions that defines what the search can reason about.
  • Duplicate-state detection: Avoids re-expanding the same state through different paths, which is often the difference between feasible and impossible search.

3.2 Adversarial search and games

Learn:

  • Minimax: Chooses moves assuming the opponent will also choose the best reply.
  • Alpha-beta pruning: Skips branches that cannot change the minimax decision, preserving the answer while saving work.
  • Negamax: Rewrites two-player zero-sum search with one symmetric value function.
  • Expectimax: Handles random events or stochastic opponents by averaging possible outcomes.
  • Iterative deepening: Searches deeper and deeper game trees, keeping the best move available if time runs out.
  • Transposition tables: Cache positions reached by different move orders so the game tree behaves more like a graph.
  • Monte Carlo Tree Search: Builds a search tree by simulating many possible continuations and focusing on promising moves.
  • UCT: Selects MCTS branches by balancing high average reward with under-explored actions.
  • Counterfactual regret minimization: Learns strategies by reducing regret for not having chosen alternatives at decision points.
  • Self-play: Improves a strategy by training or evaluating it against versions of itself.

Minimax

V(s) = max_a V(T(s, a)) for max player; min_a V(T(s, a)) for min player

UCT selection

UCT = Q(s, a) + c * sqrt(ln N(s) / N(s, a))

3.3 Constraint satisfaction and satisfiability

Study:

  • Constraint-satisfaction problems: Assign values to variables while satisfying rules such as all-different, ordering, or capacity constraints.
  • Backtracking: Builds an assignment step by step and retreats when a choice violates constraints.
  • Forward checking: Removes values that would become impossible after the current assignment.
  • Constraint propagation: Pushes implications of constraints through the problem before or during search.
  • Arc consistency: Ensures every value for one variable has at least one compatible value in a connected variable.
  • AC-3: A standard algorithm for repeatedly enforcing arc consistency across binary constraints.
  • Minimum remaining values: Chooses the variable with the fewest legal values to expose conflicts early.
  • Degree heuristic: Chooses the variable involved in many constraints because it can reduce the problem fastest.
  • Least-constraining value: Tries the value that leaves the most options open for later variables.
  • Min-conflicts: Repairs a complete but inconsistent assignment by changing variables that cause conflicts.
  • Boolean satisfiability: Asks whether true/false assignments can make a logical formula true.
  • DPLL: Solves SAT with recursive splitting plus simplifications such as unit propagation.
  • Conflict-driven clause learning: Learns new clauses from contradictions so the solver does not repeat the same mistake.
  • WalkSAT: Uses local search over truth assignments to quickly find satisfying assignments in many practical cases.
  • Integer programming: Optimizes an objective while decision variables must take integer values.
  • Mixed-integer programming: Combines integer and continuous variables for scheduling, routing, allocation, and planning tasks.

These patterns are useful in scheduling, verification, program synthesis, planning, and structured generation.

3.4 Logic and symbolic reasoning

Learn:

  • Propositional logic: Represents facts as true or false statements connected by operators such as and, or, and not.
  • First-order predicate logic: Adds objects, relations, variables, and quantified statements about a domain.
  • Quantifiers: Express whether a statement applies to all objects or at least one object.
  • Unification: Finds substitutions that make two logical expressions match.
  • Resolution: Proves statements by showing that denying the conclusion leads to contradiction.
  • Forward chaining: Starts from known facts and applies rules until new conclusions appear.
  • Backward chaining: Starts from a goal and works backward to see which facts would prove it.
  • Horn clauses: Rule-like logical clauses that support efficient inference in many expert systems.
  • Logic programming: Describes relationships and rules, then lets an inference engine answer queries.
  • Description logic: Represents classes, properties, and relationships behind many ontology systems.
  • Knowledge bases: Store facts and rules in a form that inference systems can query.
  • Ontologies: Define shared concepts and relationships so systems agree on what terms mean.
  • Non-monotonic reasoning: Allows conclusions to be withdrawn when new evidence appears.
  • Default logic: Handles usual assumptions, such as treating something as true unless contradicted.
  • Temporal logic: Reasons about time, order, and eventual or always conditions.
  • Modal logic: Reasons about necessity, possibility, belief, knowledge, or obligation.
  • Fuzzy logic: Represents graded truth rather than only true or false.
  • Automated theorem proving: Uses algorithms to prove mathematical or logical statements from axioms.

Core inference pattern

P, P => Q :: Q

Also distinguish:

  • Deduction: rule plus fact produces a necessary conclusion.
  • Induction: observations support a general rule.
  • Abduction: infer the best explanation.
  • Analogy: transfer structure from a similar case.
  • Counterfactual reasoning: determine what would happen under a different intervention.

Wikipedia's AI outline, glossary, and algorithm index are useful breadth-first references for these older and newer branches of AI.

3.5 Planning

Study:

  • State-space planning: Searches through possible world states created by applying actions.
  • STRIPS: Describes actions with preconditions plus add and delete effects.
  • PDDL: A standard planning language for defining domains, actions, objects, and goals.
  • Forward and backward planning: Forward planning starts from the initial state; backward planning regresses from the goal.
  • Partial-order planning: Builds plans where only necessary action orderings are fixed.
  • Graphplan: Uses a planning graph to alternate reachable facts and actions across time steps.
  • Hierarchical task networks: Decompose high-level tasks into smaller executable subtasks.
  • Contingent planning: Creates plans with branches for different observations or outcomes.
  • Planning under uncertainty: Chooses actions when outcomes, observations, or world state are not fully known.
  • Temporal planning: Handles action durations, deadlines, concurrency, and time windows.
  • Multi-agent planning: Coordinates actions among multiple agents with shared or competing goals.
  • Receding-horizon planning: Plans a few steps ahead, executes the first step, then replans with updated information.

Modern agent systems often rediscover these ideas under names such as planner-executor, workflow orchestration, and task decomposition.

Learning target

You should be able to model a problem as states and transitions, choose an appropriate search strategy, explain its complexity, reason about whether it is complete or optimal, implement basic game-playing search with evaluation functions, recognize when a task should be framed as a constraint or satisfiability problem, distinguish the main symbolic reasoning patterns, and connect classical planning ideas to modern agent workflows.
