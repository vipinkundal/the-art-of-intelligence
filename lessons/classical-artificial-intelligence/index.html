Phase 3

3. Classical Artificial Intelligence

Classical AI should not be skipped. It supplies many of the reasoning and planning mechanisms now being combined with language models and agents.

3.1 Search algorithms

Learn and implement:

  • Breadth-first search
  • Depth-first search
  • Depth-limited search
  • Iterative deepening
  • Uniform-cost search
  • Dijkstra's algorithm
  • Bidirectional search
  • Greedy best-first search
  • A*
  • Weighted A*
  • Iterative Deepening A*
  • Beam search
  • Branch and bound
  • Hill climbing
  • Random-restart hill climbing
  • Simulated annealing
  • Tabu search
  • Local beam search

A* evaluation

f(n) = g(n) + h(n)

where g(n) is the path cost so far and h(n) estimates the remaining cost.

Understand

  • Admissible heuristics
  • Consistent heuristics
  • Completeness
  • Optimality
  • Time and memory complexity
  • State-space representation
  • Duplicate-state detection

3.2 Adversarial search and games

Learn:

  • Minimax
  • Alpha-beta pruning
  • Negamax
  • Expectimax
  • Iterative deepening
  • Transposition tables
  • Monte Carlo Tree Search
  • UCT
  • Counterfactual regret minimization
  • Self-play

Minimax

V(s) = max_a V(T(s, a)) for max player; min_a V(T(s, a)) for min player

UCT selection

UCT = Q(s, a) + c * sqrt(ln N(s) / N(s, a))

3.3 Constraint satisfaction and satisfiability

Study:

  • Constraint-satisfaction problems
  • Backtracking
  • Forward checking
  • Constraint propagation
  • Arc consistency
  • AC-3
  • Minimum remaining values
  • Degree heuristic
  • Least-constraining value
  • Min-conflicts
  • Boolean satisfiability
  • DPLL
  • Conflict-driven clause learning
  • WalkSAT
  • Integer programming
  • Mixed-integer programming

These patterns are useful in scheduling, verification, program synthesis, planning, and structured generation.

3.4 Logic and symbolic reasoning

Learn:

  • Propositional logic
  • First-order predicate logic
  • Quantifiers
  • Unification
  • Resolution
  • Forward chaining
  • Backward chaining
  • Horn clauses
  • Logic programming
  • Description logic
  • Knowledge bases
  • Ontologies
  • Non-monotonic reasoning
  • Default logic
  • Temporal logic
  • Modal logic
  • Fuzzy logic
  • Automated theorem proving

Core inference pattern

P, P => Q :: Q

Also distinguish:

  • Deduction: rule plus fact produces a necessary conclusion.
  • Induction: observations support a general rule.
  • Abduction: infer the best explanation.
  • Analogy: transfer structure from a similar case.
  • Counterfactual reasoning: determine what would happen under a different intervention.

Wikipedia's AI outline, glossary, and algorithm index are useful breadth-first references for these older and newer branches of AI.

3.5 Planning

Study:

  • State-space planning
  • STRIPS
  • PDDL
  • Forward and backward planning
  • Partial-order planning
  • Graphplan
  • Hierarchical task networks
  • Contingent planning
  • Planning under uncertainty
  • Temporal planning
  • Multi-agent planning
  • Receding-horizon planning

Modern agent systems often rediscover these ideas under names such as planner-executor, workflow orchestration, and task decomposition.

Learning target

You should be able to model a problem as states and transitions, choose an appropriate search strategy, explain its complexity, reason about whether it is complete or optimal, implement basic game-playing search with evaluation functions, recognize when a task should be framed as a constraint or satisfiability problem, distinguish the main symbolic reasoning patterns, and connect classical planning ideas to modern agent workflows.
