Phase 15

15. Computer Vision

Computer vision covers classical image processing, recognition, detection, segmentation, geometry, reconstruction, and modern visual generation.

Vision pipeline

Pixels to features Filter, transform, and describe image structure such as edges, corners, and texture.
Features to meaning Classify, detect, segment, track, and estimate pose or depth.
Meaning to scenes Reconstruct 3D geometry, render new views, or generate visual worlds.

15.1 Classical vision

Learn:

  • Image formation: Describes how light, lenses, sensors, and projection create images.
  • Colour spaces: Represent color in systems such as RGB, HSV, LAB, or YCbCr for different tasks.
  • Histograms: Summarize pixel intensity or color distributions.
  • Convolution and filtering: Apply kernels to enhance, smooth, detect, or transform image patterns.
  • Gaussian blur: Smooths images by averaging nearby pixels with Gaussian weights.
  • Median filtering: Removes impulse noise by replacing pixels with neighborhood medians.
  • Edge detection: Finds sharp intensity changes that often mark object boundaries.
  • Canny detector: A multi-step edge detector using smoothing, gradients, suppression, and thresholds.
  • Hough transform: Detects geometric shapes such as lines and circles through parameter voting.
  • Morphological operations: Use shape-based operations like erosion and dilation on binary or grayscale images.
  • Fourier transforms: Analyze images by frequency content instead of spatial pixels.
  • Wavelets: Represent image details at multiple locations and scales.
  • Image pyramids: Store images at multiple resolutions for scale-aware processing.
  • Harris corners: Detect points where image gradients change strongly in two directions.
  • SIFT: Detects and describes scale-invariant local features for matching.
  • SURF: A faster feature detector and descriptor inspired by SIFT.
  • ORB: Efficient binary features useful for real-time matching.
  • HOG: Describes local gradient orientations, historically useful for object detection.
  • RANSAC: Fits models robustly by repeatedly sampling and rejecting outliers.
  • Optical flow: Estimates pixel or feature motion between video frames.
  • Epipolar geometry: Describes geometric constraints between two camera views.
  • Camera calibration: Estimates camera intrinsics and distortion so geometry is measurable.
  • Homographies: Map points between planar views or camera perspectives.
  • Stereo vision: Uses two views to estimate depth from disparity.
  • Structure from motion: Reconstructs camera motion and 3D structure from multiple images.

15.2 Deep vision

Study:

  • Image classification: Assigns one or more labels to an entire image.
  • Object detection: Finds object classes and bounding boxes.
  • Semantic segmentation: Assigns a class label to every pixel.
  • Instance segmentation: Separates individual object instances at pixel level.
  • Panoptic segmentation: Combines semantic and instance segmentation into one scene map.
  • Keypoint detection: Locates important points such as joints, corners, or landmarks.
  • Pose estimation: Infers body, object, or camera pose from visual evidence.
  • Tracking: Maintains object identities across video frames.
  • Depth estimation: Predicts distance from the camera for pixels or regions.
  • Optical flow: Estimates dense motion between frames using learned or classical methods.
  • Image restoration: Removes degradation such as blur, noise, compression, or damage.
  • Super-resolution: Generates a higher-resolution image from a lower-resolution input.
  • Image generation: Produces new images from noise, text, images, or structured controls.

Important families:

  • R-CNN: Classifies proposed regions to detect objects.
  • Fast R-CNN: Shares convolutional features across region proposals for faster detection.
  • Faster R-CNN: Adds a neural region proposal network.
  • YOLO: Performs single-stage real-time object detection.
  • SSD: Detects objects from multiple feature-map scales in one pass.
  • RetinaNet: Uses focal loss to handle class imbalance in dense detection.
  • DETR: Frames object detection as set prediction with transformers.
  • FCN: Fully convolutional network for dense pixel prediction.
  • U-Net: Encoder-decoder segmentation network with skip connections.
  • Mask R-CNN: Extends Faster R-CNN with instance mask prediction.
  • Vision Transformer: Processes image patches as transformer tokens.
  • Swin Transformer: Uses shifted local windows for efficient hierarchical vision transformers.
  • Segment-anything-style models: Use promptable segmentation to create masks from points, boxes, or text-like cues.
  • Masked autoencoders: Learn visual representations by reconstructing hidden image patches.

15.3 Three-dimensional vision and generation

Learn:

  • Point clouds: Represent 3D scenes as unordered sets of points in space.
  • Voxel methods: Represent 3D space as a grid of volume cells.
  • Meshes: Represent surfaces with vertices, edges, and faces.
  • Signed distance functions: Store distance to the nearest surface, with sign indicating inside or outside.
  • Neural radiance fields: Learn a continuous scene function that renders novel camera views.
  • 3D Gaussian splatting: Represents scenes with many learned Gaussian blobs that can be rendered efficiently.
  • Novel-view synthesis: Generates images of a scene from viewpoints not directly captured.
  • Multiview geometry: Uses relationships between camera views to infer depth and structure.
  • Neural rendering: Uses neural networks to synthesize images from scene representations.
  • 3D reconstruction: Builds 3D structure from images, video, scans, or learned priors.
  • Text-to-3D: Generates 3D assets or scenes from language prompts.
  • Video generation: Produces temporally coherent frame sequences.
  • World simulation: Models dynamic scenes so agents can predict and interact with environments.

Learning target

You should be able to connect classical image-processing tools to modern deep vision tasks, choose appropriate model families for recognition or geometry problems, and explain how 3D vision methods represent and render scenes.
