6. Deep Learning Foundations

Deep learning foundations cover neural network computation, convolutional architectures, and sequence models that underpin modern vision, speech, language, and multimodal systems.

Deep learning stack

Compute graph Layers transform data and automatic differentiation sends gradients backward.
Architecture Convolutions, recurrence, attention, normalization, and residual paths shape learning.
Generalize Regularization, augmentation, clipping, and distillation improve robustness and transfer.

6.1 Neural networks

Study:

  • Perceptrons: Simple linear classifiers that combine weighted inputs and a threshold.
  • Multilayer perceptrons: Stacked fully connected layers that learn nonlinear functions.
  • Computational graphs: Represent operations and dependencies so gradients can be computed automatically.
  • Backpropagation: Applies the chain rule backward through the graph to compute parameter gradients.
  • Activation functions: Add nonlinearity so networks can model more than linear transformations.
  • Weight initialization: Chooses starting weights that keep signals and gradients at useful scales.
  • Normalization: Stabilizes activations or weights to make training faster and more reliable.
  • Residual connections: Add skip paths so layers can learn refinements and gradients can flow more easily.
  • Regularization: Discourages overfitting by constraining parameters, activations, or training behavior.
  • Universal approximation: Shows neural networks can approximate broad classes of functions with enough capacity.
  • Representation learning: Learns internal features that make downstream prediction or generation easier.

Important activations

ReLU ReLU(x) = max(0, x)
GELU GELU(x) = x Phi(x)
SiLU SiLU(x) = x sigma(x)

Layer normalization

LN(x) = gamma * ((x - mu) / sqrt(sigma^2 + epsilon)) + beta

Learn

  • Batch normalization: Normalizes activations using batch statistics to stabilize training.
  • Layer normalization: Normalizes features within each example, common in transformers.
  • RMSNorm: Normalizes by root mean square without centering, often cheaper than layer normalization.
  • Dropout: Randomly disables activations during training so the model does not rely too heavily on one path.
  • Weight decay: Penalizes large weights to encourage simpler solutions.
  • Early stopping: Stops training when validation performance stops improving.
  • Data augmentation: Creates transformed training examples to improve robustness.
  • Gradient clipping: Caps gradient size to prevent unstable updates.
  • Label smoothing: Softens target labels so the model is less overconfident.
  • Knowledge distillation: Trains a smaller model to imitate a stronger teacher model.

6.2 Convolutional neural networks

Understand:

  • Discrete convolution: Slides a small kernel over data to detect local patterns.
  • Cross-correlation: The operation most deep learning libraries call convolution, without flipping the kernel.
  • Padding: Adds border values so output size and edge behavior can be controlled.
  • Stride: Moves the kernel by larger steps to reduce spatial resolution.
  • Dilation: Spreads kernel points apart to expand context without adding many parameters.
  • Pooling: Downsamples features by summarizing local regions.
  • Receptive fields: Describe how much input context can influence a feature.
  • Translation equivariance: A shifted input produces shifted features, useful for images and signals.
  • Depthwise and separable convolutions: Reduce compute by splitting spatial and channel mixing.
  • Residual networks: Use skip connections to train very deep convolutional models.
  • Encoder-decoder networks: Compress inputs and then reconstruct structured outputs.
  • U-Net architectures: Combine encoder-decoder structure with skip connections for dense prediction.

Two-dimensional convolution

Y[i, j] = sum_m sum_n X[i + m, j + n] K[m, n]

Important architectures and ideas

  • LeNet: Early convolutional network for handwritten digit recognition.
  • AlexNet: Popularized deep CNNs on GPUs for large-scale image classification.
  • VGG: Demonstrated strong results with simple stacks of small convolutions.
  • Inception: Uses parallel convolution paths to capture patterns at multiple scales.
  • ResNet: Introduced residual blocks that made very deep networks practical.
  • DenseNet: Connects each layer to many later layers to encourage feature reuse.
  • EfficientNet: Scales network depth, width, and resolution together.
  • MobileNet: Uses efficient convolutions for mobile and edge deployment.
  • U-Net: A segmentation architecture that combines local detail with high-level context.

6.3 Recurrent and sequence models

Study:

  • Recurrent neural networks: Process sequences by updating a hidden state over time.
  • Backpropagation through time: Unrolls recurrent computation across steps to compute gradients.
  • Vanishing and exploding gradients: Long sequences can shrink or amplify gradients, making training unstable.
  • LSTM: Uses gates and memory cells to preserve information over longer spans.
  • GRU: A simpler gated recurrent unit that controls update and reset behavior.
  • Bidirectional RNNs: Read sequences forward and backward when full context is available.
  • Sequence-to-sequence models: Map one sequence to another, such as translation or summarization.
  • Encoder-decoder models: Encode an input sequence into representations and decode an output sequence.
  • Attention: Lets the decoder focus on relevant input positions instead of relying on one fixed vector.
  • Connectionist temporal classification: Trains sequence labels when exact input-output alignment is unknown.

Basic recurrence

h_t = phi(W_h h_{t-1} + W_x x_t + b)

RNNs remain important conceptually even though transformers and state-space models dominate many modern sequence applications.

Learning target

You should be able to explain feed-forward networks, activation and normalization choices, convolutional architectures, and recurrent sequence models, then connect those ideas to modern deep learning systems.
