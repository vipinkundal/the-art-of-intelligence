7. Transformers And Foundation Models

The transformer remains the central architecture for language and many multimodal systems. The Wikipedia transformer article is useful for an introductory structural overview, while original papers and current courses should be used for implementation details.

Foundation model pipeline

Represent Tokenize inputs, embed them, and add position or structure information.
Train and align Pretrain on prediction tasks, then adapt with instructions, preferences, or rewards.
Serve Use efficient attention, KV caches, decoding controls, and verifiers at inference.

7.1 Transformer fundamentals

Learn:

  • Token embeddings: Learned vectors that turn discrete tokens into continuous model inputs.
  • Positional encoding: Adds order information so the model can distinguish token positions.
  • Sinusoidal positions: Fixed wave-based position features used in the original transformer.
  • Rotary positional embeddings: Rotate query and key features so attention depends on relative positions.
  • ALiBi: Adds linear attention biases that favor nearby tokens and can support length extrapolation.
  • Queries, keys and values: Attention compares queries to keys, then mixes the matching values.
  • Self-attention: Lets tokens in one sequence exchange information with each other.
  • Cross-attention: Lets one sequence attend to another, such as decoder tokens attending to encoder outputs.
  • Multi-head attention: Runs several attention heads in parallel so different relationships can be captured.
  • Masked causal attention: Prevents tokens from seeing future tokens during autoregressive generation.
  • Feed-forward networks: Per-token MLP blocks that transform features after attention mixes context.
  • Residual streams: Carry information across layers through skip connections.
  • Pre-norm and post-norm: Place normalization before or after sublayers, affecting training stability.
  • Encoder-only models: Build bidirectional representations, common for classification and retrieval encoders.
  • Decoder-only models: Generate autoregressively, common for modern language models.
  • Encoder-decoder models: Encode an input sequence and decode an output sequence, common for translation and summarization.

Scaled dot-product attention

Attention(Q, K, V) = softmax((QK^T) / sqrt(d_k)) V

Multi-head attention

MHA(Q, K, V) = Concat(head_1, ..., head_h) W^O

Autoregressive factorization

p(x_1:T) = product_{t=1..T} p(x_t | x_{<t})

Perplexity

PPL = exp(-(1 / T) * sum_{t=1..T} log p(x_t | x_{<t}))

7.2 Tokenization

Learn:

  • Character tokenization: Splits text into characters, simple but often inefficient for long contexts.
  • Word tokenization: Splits by words, intuitive but brittle for rare words and multilingual text.
  • Byte-pair encoding: Learns frequent subword merges to balance vocabulary size and sequence length.
  • WordPiece: Builds subword units that improve likelihood under a language-model objective.
  • Unigram language-model tokenization: Chooses a probabilistic vocabulary of subwords and tokenizes by likelihood.
  • Byte-level tokenization: Represents text through bytes so every string can be encoded.
  • Vocabulary construction: Chooses which tokens exist and therefore what the model can represent efficiently.
  • Unicode normalization: Standardizes visually similar text forms before tokenization.
  • Special tokens: Mark boundaries, roles, padding, tools, images, or control instructions.
  • Token fertility: Measures how many tokens are needed for a word or phrase, affecting cost and fairness across languages.
  • Multilingual tokenization: Balances vocabulary coverage across scripts and languages.
  • Tokenizer security and prompt-boundary problems: Tokenization can blur user, system, and data boundaries if controls are not explicit.

Implementing BPE from scratch is an excellent foundational project.

7.3 Efficient and alternative attention

Study:

  • Multi-query attention: Shares keys and values across heads to reduce cache memory.
  • Grouped-query attention: Shares keys and values across groups of query heads for a quality-efficiency trade-off.
  • Sparse attention: Attends to a subset of token pairs instead of all pairs.
  • Sliding-window attention: Restricts attention to nearby tokens for efficient long sequences.
  • Block-sparse attention: Uses structured blocks of allowed attention for faster computation.
  • Linear attention: Rewrites attention to scale approximately linearly with sequence length.
  • Local-global attention: Combines local neighborhoods with selected global tokens.
  • FlashAttention: Computes exact attention with less high-bandwidth memory movement.
  • Paged attention: Manages KV cache memory in pages for efficient serving.
  • KV-cache management: Stores previous keys and values so generation does not recompute old context.
  • Attention sinks: Tokens that attract attention and can stabilize long-context behavior.
  • Long-context methods: Techniques that extend usable context length through architecture, training, or retrieval.
  • Context parallelism: Splits long context processing across devices.

FlashAttention is an exact, IO-aware attention algorithm designed to reduce expensive memory movement rather than merely reducing arithmetic operations.

7.4 Pretraining

Understand:

  • Causal language modelling: Predicts the next token from previous tokens.
  • Masked language modelling: Predicts hidden tokens from surrounding context.
  • Prefix language modelling: Conditions on a visible prefix and predicts the continuation.
  • Span corruption: Masks spans of text and trains the model to reconstruct them.
  • Denoising objectives: Corrupt inputs and train the model to recover clean data.
  • Next-sentence and sentence-order objectives: Train models to reason about relationships between text segments.
  • Data mixtures: Balance sources such as web, code, math, books, dialogue, and domain data.
  • Data filtering: Removes low-quality, unsafe, duplicate, or irrelevant examples.
  • Deduplication: Reduces repeated examples that can waste compute or increase memorization.
  • Contamination: Occurs when benchmark or evaluation data leaks into training.
  • Curriculum ordering: Controls data order or difficulty over training.
  • Synthetic data: Uses generated examples to add coverage, supervision, or practice tasks.
  • Scaling laws: Empirical relationships between model size, data, compute, and loss.
  • Compute-optimal training: Balances model size and data size for a fixed compute budget.
  • Checkpointing and recovery: Saves model and optimizer state so long training runs can resume.

Empirical scaling law

L(N, D) ~= L_infinity + aN^{-alpha} + bD^{-beta}

N represents model size and D represents training data. These are empirical relationships, not universal physical laws.

7.5 Post-training

This is one of the largest areas missing from older roadmaps.

Study:

  • Continued pretraining: Keeps training the base objective on new or domain-specific data.
  • Domain adaptation: Specializes a model for a field such as law, medicine, code, or finance.
  • Supervised fine-tuning: Trains on input-output examples that demonstrate desired behavior.
  • Instruction tuning: Fine-tunes on tasks phrased as instructions so the model follows user requests better.
  • Rejection-sampling fine-tuning: Samples outputs, filters or ranks them, then fine-tunes on the better examples.
  • Reward modelling: Trains a model to score outputs according to human or AI preferences.
  • Preference modelling: Learns from comparisons such as chosen versus rejected responses.
  • RLHF: Uses human feedback and reinforcement learning to align outputs with preferences.
  • RLAIF: Uses AI feedback instead of, or alongside, human feedback for preference signals.
  • PPO-based post-training: Uses policy-gradient reinforcement learning to optimize a reward model.
  • Direct Preference Optimization: Optimizes preference pairs directly without a separate RL loop.
  • Identity Preference Optimization: A preference-optimization variant focused on preserving desired model identity or style behavior.
  • Kahneman-Tversky Optimization: A preference objective inspired by asymmetric treatment of desirable and undesirable examples.
  • Odds-ratio preference optimization: Optimizes preference odds between chosen and rejected responses.
  • Group-relative policy optimization: Compares sampled outputs within a group to create relative training signals.
  • Reinforcement learning with verifiable rewards: Uses tasks where correctness can be checked automatically.
  • Process reward models: Score intermediate reasoning steps, not only final answers.
  • Outcome reward models: Score the final result regardless of the path used to produce it.
  • Constitutional approaches: Use written principles to critique, revise, or judge model outputs.
  • Distillation from stronger models: Trains smaller or cheaper models to imitate stronger systems.
  • Online versus offline preference learning: Online methods collect feedback during deployment; offline methods train from fixed preference datasets.

Reward-model loss

L_RM = -log sigma(r_theta(x, y^+) - r_theta(x, y^-))

DPO optimizes preference data without fitting a separate reward model and then running a conventional RL loop.

RLHF has its own Wikipedia overview, but use the original papers and current post-training literature for implementation details.

7.6 Parameter-efficient adaptation

Learn:

  • Adapters: Add small trainable modules while keeping most base weights frozen.
  • LoRA: Learns low-rank weight updates instead of fine-tuning full matrices.
  • QLoRA: Combines low-rank adaptation with quantized base weights to save memory.
  • Prefix tuning: Learns prefix vectors that steer model behavior without changing core weights.
  • Prompt tuning: Learns soft prompt embeddings for a task.
  • P-tuning: Uses trainable prompt representations, often with deeper prompt encoders.
  • Low-rank and sparse updates: Restrict trainable changes to compact or selective parameter subsets.
  • Partial fine-tuning: Updates only selected layers, heads, or modules.
  • Quantization-aware adaptation: Adapts models while accounting for low-precision weights or activations.

LoRA

W' = W + (alpha / r) BA

BA is a trainable low-rank update and the original W can remain frozen.

7.7 Decoding and inference algorithms

Learn:

  • Greedy decoding: Always selects the highest-probability next token.
  • Beam search: Keeps several high-scoring partial sequences while decoding.
  • Temperature sampling: Adjusts randomness by sharpening or flattening token probabilities.
  • Top-k sampling: Samples only from the k most likely next tokens.
  • Top-p, or nucleus, sampling: Samples from the smallest token set whose probability mass reaches p.
  • Typical sampling: Prefers tokens whose surprise is close to expected surprise.
  • Contrastive decoding: Chooses tokens that are likely under a strong model and less favored by a weaker reference.
  • Diverse beam search: Encourages beams to explore different output options.
  • Constrained decoding: Forces outputs to satisfy required words, structure, or rules.
  • Grammar-constrained decoding: Restricts generation to a formal grammar such as JSON or SQL syntax.
  • Speculative decoding: Uses a small draft model and a larger verifier model to speed generation.
  • Parallel decoding: Predicts or validates multiple tokens or branches at once.
  • Best-of-N: Generates multiple candidates and selects the best by a scorer or verifier.
  • Reranking: Reorders generated candidates after decoding.
  • Self-consistency: Samples several reasoning paths and picks the most consistent answer.
  • Verifier-guided search: Uses a verifier model or rule checker to guide decoding decisions.

Temperature

p_i = exp(z_i / T) / sum_j exp(z_j / T)

Lower T sharpens the distribution; higher T increases diversity.

Learning target

You should be able to explain transformer blocks, tokenization choices, attention variants, pretraining objectives, post-training methods, parameter-efficient adaptation, and decoding strategies used by modern foundation models.
