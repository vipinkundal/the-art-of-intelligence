Phase 20

20. Graph Machine Learning

Graph machine learning studies models that learn from nodes, edges, neighborhoods, paths, communities, and knowledge-graph relations instead of treating examples as independent rows.

Graph learning workflow

Represent structure Encode entities as nodes, relationships as edges, and neighborhoods as graph context.
Propagate information Aggregate messages from neighbors using random walks, spectral filters, or neural layers.
Predict on graphs Classify nodes, edges, whole graphs, links, communities, or retrieved knowledge.

Study

  • Graph terminology: Learn nodes, edges, degree, paths, walks, components, directed graphs, weighted graphs, bipartite graphs, and heterogeneous graphs.
  • Adjacency and Laplacian matrices: Represent graph connectivity with matrices used by spectral methods, diffusion, and graph neural networks.
  • Spectral graph theory: Uses eigenvalues and eigenvectors of graph operators to study smoothness, cuts, clustering, and convolution-like filters.
  • Random walks: Sample local graph neighborhoods and define transition processes used by ranking, embeddings, and diffusion methods.
  • PageRank: Scores nodes by stationary importance under a random-surfer model with teleportation.
  • Community detection: Finds groups of nodes that are more densely connected internally than externally.
  • Graph embeddings: Map nodes, edges, or whole graphs into vectors that preserve useful structure for downstream models.
  • Knowledge graphs: Represent typed facts as entities and relations, often as triples such as subject, relation, object.
  • Link prediction: Predicts missing or future edges from topology, node features, relation types, or learned embeddings.
  • Node classification: Predicts labels for individual nodes using their features and graph neighborhood context.
  • Edge classification: Predicts labels or relation types for edges, such as transaction risk or semantic relation category.
  • Graph classification: Predicts labels for whole graphs, such as molecule property, program behavior, or document structure.
  • Message-passing neural networks: Update each node by aggregating transformed information from neighboring nodes and edges.

Core matrices and scores

Adjacency A_ij = 1 if edge i -> j exists, otherwise 0
Graph Laplacian L = D - A
Normalized Laplacian L_norm = I - D^(-1/2) A D^(-1/2)
PageRank r = alpha P^T r + (1 - alpha) v

Important algorithms

  • PageRank: Ranks pages, papers, accounts, or entities by recursive importance in a link graph.
  • Node2vec: Learns node embeddings from biased random walks that trade off breadth-first and depth-first exploration.
  • DeepWalk: Learns node embeddings by treating random walks like sentences and optimizing a skip-gram objective.
  • Graph convolutional networks: Apply neighborhood averaging and feature transforms to learn node representations.
  • GraphSAGE: Samples and aggregates neighbors so graph neural networks can scale and generalize to unseen nodes.
  • Graph attention networks: Learn attention weights over neighbors instead of treating all neighbor messages equally.
  • Graph isomorphism networks: Use expressive aggregation to distinguish graph structures more strongly than basic GCN layers.
  • Relational GCN: Extends graph convolution to multi-relation graphs such as knowledge graphs.
  • Graph transformers: Use attention over graph tokens, often with structural encodings for distance, paths, or edge types.
  • TransE: Learns knowledge-graph embeddings where head plus relation vector should land near the tail vector.
  • Graph-based retrieval: Uses graph neighborhoods, paths, communities, or knowledge links to retrieve evidence for search and RAG systems.

Generic message passing

Aggregate m_v^(l) = AGG({ phi(h_u^(l), h_v^(l), e_uv) : u in N(v) })
Update h_v^(l+1) = psi(h_v^(l), m_v^(l))

The main design choice is what counts as a message, how messages are aggregated, and how the node state is updated without losing useful structural information.

Where it connects now

Graph ML increasingly intersects with foundation models, retrieval, and agent systems. Knowledge graphs can ground retrieval, graph embeddings can improve recommendations and search, and graph neural networks can reason over molecules, social networks, fraud networks, code, traffic, documents, and structured memory.

Learning target

You should be able to represent graph data with adjacency and Laplacian matrices, explain random-walk and spectral methods, compare PageRank, Node2vec, DeepWalk, GCN, GraphSAGE, GAT, GIN, R-GCN, graph transformers, and knowledge-graph embeddings, and choose graph-learning methods for node, edge, graph, link-prediction, retrieval, and knowledge-graph tasks.
