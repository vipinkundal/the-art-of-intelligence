22. Interpretability And Mechanistic Understanding

Interpretability studies how to explain model behavior, inspect neural representations, and identify internal mechanisms instead of treating an AI system as a black box.

Learn three levels

Model-agnostic explanations Explain predictions from the outside using perturbations, surrogates, importance scores, and counterfactuals.
Neural-network interpretation Inspect gradients, saliency, activations, probes, features, and representation geometry.
Mechanistic understanding Find circuits, causal pathways, features, and computational structures inside trained models.

Model-agnostic interpretation

These methods can be applied to many model families because they mostly treat the model as a prediction function and examine how outputs change when inputs change.

  • Permutation importance: Measures performance drop after shuffling a feature, estimating how much the model relies on that feature.
  • Partial dependence plots: Show average model predictions as one or two features vary across a chosen range.
  • Individual conditional expectation: Shows feature-response curves for individual examples instead of only the population average.
  • LIME: Fits a simple local surrogate model around one prediction to approximate nearby model behavior.
  • SHAP: Assigns feature contributions using Shapley-value ideas from cooperative game theory.
  • Counterfactual explanations: Finds small input changes that would flip or meaningfully change a prediction.
  • Surrogate models: Trains an interpretable model to mimic a more complex model globally or locally.

Neural-network interpretation

These methods inspect gradients, internal activations, spatial regions, learned directions, and representation spaces in neural networks.

  • Saliency maps: Highlight input dimensions that most affect a prediction according to gradients or related scores.
  • Integrated gradients: Attribute prediction change to features by accumulating gradients from a baseline to the input.
  • Grad-CAM: Uses gradients flowing into convolutional feature maps to localize image regions relevant to a class.
  • Occlusion analysis: Masks or removes input regions to see which parts cause prediction changes.
  • Feature visualization: Optimizes inputs or patterns that strongly activate a neuron, channel, layer, or feature direction.
  • Representation similarity: Compares activation spaces across layers, models, checkpoints, or datasets.
  • Linear probes: Train simple classifiers on frozen activations to test what information is linearly accessible.
  • Concept activation vectors: Represent human-defined concepts as directions in activation space and test their influence.

Mechanistic interpretability

Mechanistic interpretability attempts to identify internal computational structures rather than only describing input-output correlations.

  • Activation patching: Replaces activations from one run with activations from another to test causal influence on behavior.
  • Causal tracing: Tracks where information is stored and used by intervening on hidden states across layers and positions.
  • Logit attribution: Decomposes contributions to output logits across residual streams, heads, layers, or features.
  • Attention-head analysis: Studies specialized heads, attention patterns, induction behavior, copying, retrieval, or syntax-like roles.
  • Circuit analysis: Identifies subnetworks of features, heads, MLPs, and residual paths that implement a behavior.
  • Superposition: Studies how models represent more features than dimensions by using overlapping directions.
  • Sparse autoencoders: Decompose dense activations into sparse, more interpretable feature sets.
  • Feature dictionaries: Build catalogs of learned features and their activation patterns for model inspection.
  • Automated circuit discovery: Uses algorithms to search for model components and connections that explain a behavior.
  • Model editing: Modifies model internals or parameters to change a targeted fact, behavior, or association.

Useful formulas and tests

Permutation importance I_j = score(f, X, y) - score(f, permute_j(X), y)
Integrated gradients IG_i(x) = (x_i - x'_i) integral_0^1 dF(x' + a(x - x')) / dx_i da
Local surrogate explanation(x) = argmin_g L(f, g, pi_x) + Omega(g)

Why it matters for AI

Interpretability helps teams debug failures, detect shortcut learning, audit high-impact models, compare representations, and build more trustworthy systems. Mechanistic work becomes especially important when models are too capable or too opaque to evaluate only through aggregate benchmarks.

Learning target

You should be able to compare model-agnostic explanations, neural-network attribution methods, probes, feature visualization, activation patching, causal tracing, logit attribution, circuit analysis, superposition, sparse autoencoders, automated circuit discovery, and model editing, while explaining what each method can and cannot prove.
