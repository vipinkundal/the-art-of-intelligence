Recommended learning sequence

A practical route from math to working AI systems.

For a full-stack developer, this ordering maximizes practical progress without sacrificing theory. Build each layer once, measure it, then move forward.

How to use it

Do not only read. Produce evidence.

Implement once Build core algorithms with minimal libraries before leaning on frameworks.
Measure behavior Track accuracy, cost, latency, calibration, regressions, and failure cases.
Ship artifacts Turn each stage into notes, code, experiments, and one visible project.

Eight stages

The Roadmap

  1. 01

    Stage 1

    Mathematics and classical ML

    Learn linear algebra, calculus, probability, statistics, and optimization alongside core machine-learning methods.

    • Linear regression
    • Logistic regression
    • k-means
    • PCA
    • Decision trees
    • Random forests
    • Gradient boosting
    • SVMs
    Build: Every algorithm once using only NumPy.
    Math lessons ML lessons
  2. 02

    Stage 2

    Deep-learning fundamentals

    Implement the machinery that makes neural networks trainable before treating a framework as a black box.

    • A small autodiff engine
    • MLP
    • Backpropagation
    • CNN
    • RNN or LSTM
    • Optimizers
    • Normalization
    • Training and validation loops
    Then: Reproduce a modest image-classification experiment in PyTorch.
    Deep learning Vision track
  3. 03

    Stage 3

    Transformers from scratch

    Build the complete decoder-only stack so language-model systems feel concrete, not magical.

    • BPE tokenizer
    • Token embedding
    • Causal self-attention
    • Multi-head attention
    • Transformer block
    • Decoder-only language model
    • Training loop
    • Sampling system
    • KV cache
    Outcome: A tiny language model trained, sampled, and profiled end to end.
    Transformers Attention lesson
  4. 04

    Stage 4

    Retrieval system

    Do not begin with an agent. First make retrieval measurable and reliable.

    • BM25
    • Dense embeddings
    • Hybrid retrieval
    • Cross-encoder reranking
    • Citation generation
    • Retrieval evaluation
    • Faithfulness evaluation
    Build: A RAG system with source-grounded answers and evaluation reports.
    RAG lesson Reasoning
  5. 05

    Stage 5

    Post-training

    Run small controlled comparisons before treating fine-tuning or preference optimization as defaults.

    • Prompting only
    • Supervised fine-tuning
    • LoRA
    • Preference optimization
    • Best-of-N plus a verifier
    Track: Capability improvement, forgetting, calibration, cost, and regressions.
    Foundation models Risk checks
  6. 06

    Stage 6

    Agent engineering

    Build a constrained tool-using agent with explicit boundaries, observable state, and repeatable tests.

    • Typed tool schemas
    • Explicit state
    • Planning
    • Retry limits
    • Sandboxed execution
    • Human approval for consequential actions
    • Tracing
    • A reproducible evaluation suite
    Avoid: Uncontrolled recursive autonomous-agent loops.
    Agents Tools lesson
  7. 07

    Stage 7

    AI systems

    Learn how models behave under real serving, memory, latency, and monitoring constraints.

    • Profiling
    • GPU memory calculation
    • Mixed precision
    • FlashAttention
    • Distributed data parallelism
    • Quantization
    • KV caching
    • Batching
    • Speculative decoding
    • Serving and monitoring
    Build: A small serving stack with metrics, batching, rollback notes, and cost estimates.
    Infrastructure Systems foundations
  8. 08

    Stage 8

    Choose a specialization

    Choose one primary area, reproduce two or three papers, and attempt one original extension.

    • Language models and agents
    • Computer vision and multimodality
    • Generative media
    • Reinforcement learning and robotics
    • Graph machine learning
    • Causal inference
    • Interpretability and safety
    • AI systems and compilers
    • Scientific machine learning
    Proof: A focused portfolio artifact with reproduction notes and one tested extension.
