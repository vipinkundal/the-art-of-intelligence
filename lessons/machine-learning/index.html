Phase 5

5. Classical Machine Learning

Classical machine learning covers the supervised models, objectives, losses, calibration methods, and model families that still power many reliable production systems.

Model workflow

Fit Choose a model family and objective that match the prediction problem.
Validate Use held-out data, calibration, and error analysis to check generalization.
Deploy or revise Compare metrics, costs, failure modes, and maintenance needs before shipping.

5.1 Supervised learning

Regression

Learn:

  • Ordinary least squares: Fits a linear model by minimizing squared prediction error.
  • Polynomial regression: Adds powers of input features so a linear model can fit curved relationships.
  • Ridge regression: Adds an L2 penalty to shrink weights and reduce overfitting.
  • Lasso: Adds an L1 penalty that can drive some feature weights to zero.
  • Elastic net: Combines L1 and L2 penalties for sparse but stable regression.
  • Generalized linear models: Extend linear regression to different output distributions and link functions.
  • Poisson regression: Models count data such as events per time period.
  • Quantile regression: Predicts conditional quantiles instead of only the mean.
  • Robust regression: Reduces the influence of outliers compared with ordinary squared loss.
  • Gaussian-process regression: Uses a kernel-defined distribution over functions to predict with uncertainty.

Linear regression

y_hat = Xw + b

Mean squared error

L = (1 / n) * sum_i (y_i - y_hat_i)^2

Ridge objective

min_w ||Xw - y||_2^2 + lambda ||w||_2^2

Classification

Learn:

  • Logistic regression: A linear classifier that predicts class probability with a sigmoid link.
  • Multinomial logistic regression: Extends logistic regression to more than two classes with softmax probabilities.
  • Perceptron: An early online linear classifier updated when it makes mistakes.
  • k-nearest neighbours: Predicts from the labels of the closest training examples.
  • Naive Bayes: Uses Bayes' theorem with a simplifying conditional-independence assumption.
  • Linear discriminant analysis: Classifies with Gaussian class models that share covariance.
  • Quadratic discriminant analysis: Allows each class to have its own covariance, producing curved boundaries.
  • Support-vector machines: Find a margin-maximizing decision boundary, often with hinge loss.
  • Kernel methods: Compare examples through kernels so linear models can act in richer feature spaces.
  • Decision trees: Split features into rules that form an interpretable prediction tree.
  • Random forests: Average many randomized trees to reduce variance.
  • Extremely randomized trees: Add more random splitting to decorrelate trees further.
  • AdaBoost: Builds an ensemble by focusing later learners on previously misclassified examples.
  • Gradient boosting: Adds weak learners that correct residual errors from the current ensemble.
  • XGBoost: A regularized, optimized gradient-boosted tree system widely used on tabular data.
  • LightGBM: A fast gradient-boosting library optimized for large datasets and categorical handling.
  • CatBoost: Gradient boosting with strong categorical-feature handling and ordered boosting.
  • Probability calibration: Adjusts predicted probabilities so confidence matches observed frequencies.

Sigmoid

sigma(z) = 1 / (1 + exp(-z))

Softmax

p_i = exp(z_i) / sum_j exp(z_j)

Binary cross-entropy

L = -(1 / n) * sum_i [y_i log p_i + (1 - y_i) log(1 - p_i)]

Multiclass cross-entropy

L = -sum_{c=1..C} y_c log p_c

SVM objective

min_{w,b} (1 / 2) ||w||^2 + C * sum_i max(0, 1 - y_i(w^T x_i + b))

5.2 Unsupervised learning

Clustering

Learn:

  • k-means and k-means++: Partition data around centroids; k-means++ improves centroid initialization.
  • k-medoids: Uses real data points as cluster centers, making it more robust to outliers.
  • Gaussian mixture models: Represent clusters as a weighted mixture of Gaussian distributions.
  • Expectation-maximization: Alternates between assigning soft cluster responsibility and updating parameters.
  • Agglomerative clustering: Builds clusters bottom-up by repeatedly merging nearby groups.
  • Divisive clustering: Starts with all data together and recursively splits groups apart.
  • DBSCAN: Finds dense regions and marks isolated points as noise.
  • HDBSCAN: Extends DBSCAN to handle clusters with varying density.
  • OPTICS: Orders points to reveal density-based clustering structure at multiple scales.
  • Mean shift: Moves points toward local density peaks without choosing cluster count upfront.
  • Spectral clustering: Uses graph eigenvectors to cluster data with non-convex structure.
  • Affinity propagation: Passes messages between points to choose representative exemplars.
  • BIRCH: Builds compact clustering summaries for large datasets.
  • Fuzzy c-means: Assigns each point partial membership in multiple clusters.

k-means objective

min_{mu_k} sum_i ||x_i - mu_{c_i}||_2^2

Dimensionality reduction

Learn:

  • PCA: Projects data onto directions of highest variance.
  • Kernel PCA: Applies PCA in an implicit nonlinear feature space.
  • Independent component analysis: Separates signals into statistically independent components.
  • Non-negative matrix factorization: Decomposes non-negative data into additive parts.
  • Factor analysis: Explains observed variables through fewer latent factors plus noise.
  • Random projections: Reduces dimension with random matrices while approximately preserving distances.
  • Multidimensional scaling: Places points in low dimensions to preserve pairwise distances.
  • Isomap: Preserves manifold geodesic distances rather than straight-line distances.
  • Locally linear embedding: Preserves local neighbor reconstruction relationships in a low-dimensional map.
  • t-SNE: Visualizes local neighborhoods by emphasizing nearby relationships.
  • UMAP: Builds a neighborhood graph and optimizes a low-dimensional embedding.
  • Autoencoders: Learn nonlinear compressed representations through reconstruction.

Use t-SNE and UMAP mainly for exploratory visualization; do not treat distances in their 2D plots as definitive evidence.

Other unsupervised methods

Include:

  • Topic modelling with latent Dirichlet allocation: Represents documents as mixtures of latent topics.
  • Matrix factorization: Decomposes a matrix into lower-dimensional factors, common in recommendations.
  • Collaborative filtering: Predicts preferences from user-item interaction patterns.
  • Apriori: Mines frequent itemsets by expanding only candidates whose subsets are frequent.
  • FP-growth: Mines frequent patterns with a compressed tree representation.
  • ECLAT: Mines frequent itemsets using vertical transaction-id sets.
  • Isolation forest: Detects anomalies by how easily random splits isolate them.
  • Local outlier factor: Scores points that are sparse relative to their local neighborhood.
  • One-class SVM: Learns a boundary around normal examples to detect novelty.
  • Robust covariance estimation: Estimates covariance while reducing the influence of outliers.

5.3 Evaluation and generalization

Learn:

  • Train, validation and test splits: Separate data for fitting, tuning, and final unbiased evaluation.
  • Cross-validation: Repeats training over different folds to estimate performance more reliably.
  • Stratified sampling: Preserves class proportions across splits or batches.
  • Data leakage: Occurs when training indirectly sees information that should only be available at evaluation or deployment time.
  • Overfitting and underfitting: Overfitting memorizes noise; underfitting misses real structure.
  • Regularization: Adds constraints or penalties to improve generalization.
  • Learning curves: Plot performance versus data size or training progress to diagnose bias and variance.
  • Hyperparameter optimization: Searches settings that are not learned directly by the model.
  • Grid search: Tests a fixed grid of hyperparameter combinations.
  • Random search: Samples combinations and often covers useful ranges more efficiently than grids.
  • Bayesian optimization: Uses a surrogate model to choose promising hyperparameters.
  • Successive halving: Allocates more resources to promising configurations and stops weak ones early.
  • Population-based training: Evolves hyperparameters while training multiple models in parallel.

Classification metrics

Precision Precision = TP / (TP + FP)
Recall Recall = TP / (TP + FN)
F1 F1 = 2 * Precision * Recall / (Precision + Recall)

Also understand

  • Accuracy: Fraction of predictions that are correct, best when classes are balanced.
  • Balanced accuracy: Averages recall across classes to handle imbalance.
  • Specificity: Measures how well negatives are correctly identified.
  • ROC-AUC: Measures ranking quality across true-positive and false-positive trade-offs.
  • PR-AUC: Focuses on precision-recall trade-offs, useful for rare positive classes.
  • Log loss: Penalizes confident wrong probabilities more heavily.
  • Brier score: Measures squared error of predicted probabilities.
  • Expected calibration error: Summarizes mismatch between confidence and observed accuracy.
  • Matthews correlation coefficient: A balanced classification score using all confusion-matrix cells.
  • Confusion matrices: Break predictions into true positives, false positives, true negatives, and false negatives.
  • Cost-sensitive evaluation: Scores errors according to their real-world cost, not just their count.

Regression metrics

  • MAE: Mean absolute error measures average absolute prediction mistake.
  • MSE: Mean squared error penalizes larger errors more strongly.
  • RMSE: Square root of MSE, reported in the same units as the target.
  • R^2: Measures variance explained relative to a constant baseline.
  • MAPE and its limitations: Percentage error is intuitive but unstable near zero targets.
  • Quantile loss: Evaluates predictions of a chosen quantile rather than the mean.

Ranking and retrieval metrics

  • Precision@k: Fraction of the top k results that are relevant.
  • Recall@k: Fraction of all relevant results found within the top k.
  • Mean reciprocal rank: Rewards systems that place the first relevant result early.
  • Mean average precision: Averages precision at the ranks where relevant items appear.
  • NDCG: Rewards highly relevant items appearing near the top of a ranked list.

Learning target

You should be able to choose a regression or classification model, explain its objective, recognize the loss it optimizes, compare linear, kernel, tree, ensemble, and calibrated probabilistic classifiers, select unsupervised methods for clustering, dimensionality reduction, association mining, recommendation, and anomaly detection, and evaluate models without leaking data or over-trusting a single metric.
