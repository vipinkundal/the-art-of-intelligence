14. Mixture-Of-Experts Models

Mixture-of-experts models scale parameter count by routing each token through a small subset of specialized expert networks.

Routing idea

Token Each token arrives with its current hidden representation.
Router A gating network chooses a small number of experts for that token.
Experts Selected expert outputs are combined, while inactive experts cost little compute.

Learn

  • Sparse mixture-of-experts: Activates only a small subset of expert networks per token, increasing parameters without equal compute growth.
  • Expert routing: Chooses which expert networks should process each token or example.
  • Top-k gating: Sends a token to the highest-scoring few experts rather than all experts.
  • Load balancing: Prevents the router from overusing a few experts while leaving others idle.
  • Expert capacity: Limits how many tokens an expert can handle in one batch, affecting dropped tokens and efficiency.
  • Expert parallelism: Places different experts on different devices so large expert sets can run in parallel.
  • Router instability: Routing choices can fluctuate or collapse, making training harder than dense models.
  • Auxiliary routing loss: Adds training pressure that encourages balanced and useful expert assignment.
  • Shared experts: Always-active experts provide common capability while routed experts add specialization.
  • Fine-grained experts: Use smaller or more numerous experts to give the router more precise choices.
  • Expert specialization: Experts can learn different patterns, domains, token types, or functions through routing pressure.

Generic MoE layer

y = sum_{i in TopK(g(x))} g_i(x) E_i(x)

MoE models allow parameter count to grow without activating every parameter for every token, although they introduce routing, communication, and load-balancing challenges.

Learning target

You should be able to explain how token routing, expert capacity, auxiliary load-balancing losses, and expert parallelism affect MoE model quality and serving efficiency.
