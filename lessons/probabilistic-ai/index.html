Phase 4

4. Probabilistic AI

Probabilistic AI gives models a language for uncertainty, hidden state, noisy evidence, and inference over structured variables.

Uncertainty pipeline

Structure Graphs describe which variables influence, explain, or depend on each other.
Inference Algorithms update hidden beliefs from noisy or incomplete evidence.
State over time Sequential models track changing hidden states behind observed signals.

4.1 Probabilistic graphical models

Study:

  • Bayesian networks: Directed graphs that represent probabilistic cause-effect or dependency structure between variables.
  • Markov random fields: Undirected graphs that model variables through local compatibility rather than directed causes.
  • Factor graphs: Bipartite graphs that separate variables from factor functions, making inference algorithms easier to express.
  • Conditional random fields: Discriminative graphical models that predict structured outputs conditioned on observed inputs.
  • Conditional independence: States when two variables become unrelated after another variable is known.
  • D-separation: A graph rule for reading conditional independence relationships from a Bayesian network.
  • Exact inference: Computes posterior probabilities exactly, usually only feasible for smaller or structured graphs.
  • Approximate inference: Trades exactness for tractability using sampling, optimization, or message approximations.

Important algorithms

  • Variable elimination: Sums out hidden variables one at a time to answer probability queries.
  • Sum-product algorithm: Passes messages to compute marginal probabilities efficiently on tree-structured graphs.
  • Max-product algorithm: Finds the most likely assignment instead of marginal probabilities.
  • Belief propagation: Updates local beliefs by passing messages between connected variables and factors.
  • Loopy belief propagation: Applies message passing to graphs with cycles, often useful but not always exact.
  • Junction-tree algorithm: Converts a graph into a tree of variable clusters to enable exact inference.
  • Expectation-maximization: Alternates between estimating hidden variables and updating model parameters.
  • Variational inference: Approximates a hard posterior with a simpler distribution found through optimization.
  • Gibbs sampling: Samples one variable at a time from its conditional distribution.
  • Metropolis-Hastings: Proposes random moves and accepts them with a probability that preserves the target distribution.
  • Hamiltonian Monte Carlo: Uses gradient-informed proposals to explore continuous posteriors more efficiently.
  • Sequential Monte Carlo: Tracks a distribution over time using weighted particles and resampling.

4.2 Sequential models and state estimation

Learn:

  • Markov chains: Model state sequences where the next state depends only on the current state.
  • Hidden Markov models: Add hidden states that generate observations, useful when the true state is not directly visible.
  • Forward-backward algorithm: Computes smoothed probabilities over hidden states using past and future evidence.
  • Viterbi decoding: Finds the single most likely hidden-state path through a sequence.
  • Kalman filter: Tracks a linear-Gaussian hidden state as new noisy observations arrive.
  • Extended Kalman filter: Linearizes nonlinear dynamics around the current estimate.
  • Unscented Kalman filter: Uses carefully chosen sample points to handle nonlinear transformations more accurately.
  • Particle filter: Represents hidden-state uncertainty with weighted samples instead of a single Gaussian.
  • Dynamic Bayesian networks: Extend graphical models across time steps to represent temporal dependencies.

HMM joint distribution

p(z_1:T, x_1:T) = p(z_1) * product_{t=2..T} p(z_t | z_{t-1}) * product_{t=1..T} p(x_t | z_t)

Kalman prediction

State mean x_hat_{t|t-1} = A x_hat_{t-1|t-1} + B u_t
State covariance P_{t|t-1} = A P_{t-1|t-1} A^T + Q

Learning target

You should be able to represent uncertainty with graphical models, choose an inference algorithm, explain how sequence models track hidden state over time, and recognize when filtering or smoothing is the right tool.
