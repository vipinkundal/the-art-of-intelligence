Phase 18

18. Reinforcement Learning

Reinforcement learning studies agents that learn from rewards through bandits, Markov decision processes, value learning, policy gradients, actor-critic methods, model-based RL, offline RL, and imitation.

Reinforcement learning loop

Observe state The agent reads the situation it can perceive.
Act and receive reward An action changes the environment and produces feedback.
Improve behavior Values, policies, or world models update for future choices.

18.1 Foundations

Learn:

  • Multi-armed bandits: Model repeated choices where each option has an unknown reward distribution and no long-term state.
  • Markov decision processes: Formalize sequential decisions with states, actions, transition probabilities, rewards, and discounting.
  • Partially observable MDPs: Handle settings where the agent receives observations instead of the full hidden state.
  • States, actions, rewards and transitions: Define what the agent sees, what it can do, what feedback it receives, and how the world changes.
  • Policies: Map states or observations to actions, either deterministically or as probabilities.
  • Returns: Sum future rewards so the agent optimizes long-term outcome rather than only immediate feedback.
  • Discounting: Weighs near-term rewards more than distant rewards, often making infinite-horizon problems stable.
  • Value functions: Estimate how much future return to expect from a state or state-action pair.
  • Advantage functions: Measure whether an action is better or worse than the usual value of the state.
  • Bellman equations: Express values recursively as immediate reward plus discounted future value.
  • Exploration versus exploitation: Balances trying uncertain actions against using the best-known action.

Return:

G_t = sum_{k=0..infinity} gamma^k r_{t+k+1}

Bellman expectation equation:

V^pi(s) = E_pi[r + gamma V^pi(s') | s]

Bellman optimality equation:

Q*(s, a) = E[r + gamma max_a' Q*(s', a')]

18.2 Bandits

Learn:

  • epsilon-greedy: Usually picks the best-known action but sometimes explores randomly.
  • Upper-confidence bound: Chooses actions using both estimated reward and uncertainty about that estimate.
  • Thompson sampling: Samples from a belief distribution over action quality, naturally exploring uncertain options.
  • Contextual bandits: Use side information about the current user, item, or situation before choosing an action.
  • Bayesian bandits: Maintain probability distributions over unknown rewards and update them as evidence arrives.
  • Regret analysis: Measures how much reward was lost compared with always choosing the best action in hindsight.

18.3 Tabular reinforcement learning

Study:

  • Dynamic programming: Solves MDPs by using known transition and reward models to update values exactly.
  • Policy evaluation: Computes how good a fixed policy is before trying to improve it.
  • Policy iteration: Alternates evaluating a policy and making it greedier with respect to its values.
  • Value iteration: Repeatedly applies the Bellman optimality update until values imply a strong policy.
  • Monte Carlo prediction and control: Learns from complete sampled episodes rather than a known model.
  • Temporal-difference learning: Updates estimates from partial experience using bootstrapped predictions.
  • TD(0): Updates a value estimate one step at a time using the next state's estimate.
  • TD(lambda): Blends short and long credit assignment through eligibility traces.
  • SARSA: Learns action values for the policy actually being followed, including its exploration behavior.
  • Q-learning: Learns optimal action values off-policy by bootstrapping from the best next action.
  • Eligibility traces: Keep temporary credit for recently visited states or actions so rewards can update them together.

Q-learning:

Q(s, a) <- Q(s, a) + alpha [r + gamma max_a' Q(s', a') - Q(s, a)]

18.4 Deep reinforcement learning

Value-based:

  • DQN: Uses a neural network to approximate Q-values for high-dimensional observations such as pixels.
  • Double DQN: Reduces overestimation by separating action selection from action evaluation.
  • Dueling DQN: Splits value and advantage streams so the network can learn state value separately from action preference.
  • Prioritized replay: Reuses important or surprising experiences more often during training.
  • Distributional RL: Predicts a distribution of returns instead of only the expected return.
  • Rainbow: Combines several DQN improvements into one stronger value-based agent.

Policy-based:

  • REINFORCE: Estimates policy gradients from sampled returns and directly increases probability of rewarding actions.
  • Natural policy gradient: Scales updates by the geometry of the policy distribution for more stable steps.
  • TRPO: Limits each policy update so the new policy stays close to the old one.
  • PPO: Uses a clipped objective to get much of TRPO's stability with simpler implementation.

Actor-critic:

  • A2C: Trains an actor policy with a critic value estimate in synchronized batches.
  • A3C: Runs multiple asynchronous workers so diverse experience stabilizes learning.
  • DDPG: Extends actor-critic learning to continuous actions with deterministic policies.
  • TD3: Improves DDPG with clipped double critics, delayed policy updates, and target smoothing.
  • SAC: Optimizes both reward and policy entropy, encouraging robust exploration in continuous control.

Policy-gradient theorem:

grad_theta J(theta) = E_pi_theta[grad_theta log pi_theta(a | s) Q^pi_theta(s, a)]

PPO clipped objective:

L_CLIP = E_t[min(r_t(theta) A_hat_t, clip(r_t(theta), 1 - epsilon, 1 + epsilon) A_hat_t)]

18.5 Advanced reinforcement learning

Learn:

  • Model-based RL: Learns or uses a model of environment dynamics to plan before acting.
  • Dyna: Mixes real experience with simulated experience generated from a learned model.
  • Learned world models: Predict future observations, states, or rewards so agents can imagine rollouts.
  • Model-predictive control: Plans over a short horizon, executes the first action, then replans.
  • Offline RL: Learns policies from fixed logged data without additional environment interaction.
  • Conservative Q-learning: Penalizes over-optimistic values for unseen actions in offline datasets.
  • Implicit Q-learning: Learns from offline data without explicitly maximizing over actions outside the dataset.
  • Advantage-weighted actor-critic: Imitates dataset actions more strongly when their estimated advantage is high.
  • Imitation learning: Learns behavior from demonstrations rather than reward trial and error alone.
  • Behaviour cloning: Trains a policy as supervised learning from expert state-action examples.
  • DAgger: Collects expert corrections on the learner's visited states to reduce compounding mistakes.
  • Inverse RL: Infers a reward function that would make expert behavior look rational.
  • GAIL: Uses adversarial training to make learner behavior indistinguishable from demonstrations.
  • Hierarchical RL: Learns high-level skills or subgoals that organize lower-level actions.
  • Options: Represent temporally extended actions with an initiation set, internal policy, and termination rule.
  • Meta-RL: Trains agents to adapt quickly to new tasks from limited experience.
  • Multi-agent RL: Studies learning when multiple agents interact, cooperate, or compete.
  • QMIX: Learns decentralized agent policies with a centralized value decomposition during training.
  • MADDPG: Extends actor-critic methods to multiple agents with centralized critics.
  • Self-play: Generates increasingly challenging opponents or partners from the agent's own past policies.
  • Curriculum learning: Orders tasks from easier to harder so policies build useful capabilities gradually.
  • Safe RL: Tries to maximize reward while avoiding unacceptable harm, constraint violations, or unsafe exploration.
  • Constrained MDPs: Add cost constraints to the MDP so optimization must satisfy safety or resource limits.

Learning target

You should be able to formulate an RL problem, derive the core Bellman and policy-gradient updates, choose between value-based, policy-based, actor-critic, model-based, offline, imitation, and multi-agent methods, and explain their exploration and stability tradeoffs.
