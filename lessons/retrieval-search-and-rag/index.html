8. Retrieval, Search And RAG

Retrieval-augmented generation was proposed as a way to combine a parametric language model with non-parametric external memory.

Retrieval pipeline

Index knowledge Parse, chunk, embed, and store source material so it can be searched later.
Find evidence Use lexical, dense, hybrid, and reranking methods to select relevant context.
Generate with proof Pack context into the model and return answers with citations or provenance.

8.1 Classical information retrieval

Study:

  • Inverted indexes: Map terms to the documents that contain them, making keyword lookup fast.
  • Term frequency: Measures how often a query term appears in a document.
  • Inverse document frequency: Downweights common terms and upweights rarer, more specific terms.
  • TF-IDF: Scores documents by combining local term frequency with global term rarity.
  • BM25: A strong lexical ranking function that normalizes term frequency and document length.
  • Boolean retrieval: Uses exact logical operators such as AND, OR, and NOT to match documents.
  • Tokenization and stemming: Break text into searchable units and reduce variants to shared roots.
  • Fielded search: Searches structured fields like title, author, tags, or body with different weights.
  • Query expansion: Adds related terms or reformulations to improve recall.
  • Learning to rank: Trains ranking models from relevance labels, clicks, or preference data.

TF-IDF

tfidf(t, d) = tf(t, d) log(N / df(t))

BM25

score(D, Q) = sum_{q in Q} IDF(q) * f(q, D)(k_1 + 1) / (f(q, D) + k_1(1 - b + b|D| / avgdl))

8.2 Neural retrieval

Learn:

  • Embedding models: Encode queries and documents as vectors so semantic similarity can be searched.
  • Bi-encoders: Encode query and document separately, enabling fast approximate vector search.
  • Cross-encoders: Score query-document pairs together, often more accurate but slower.
  • Late-interaction models: Keep token-level representations and compare them late for a quality-speed middle ground.
  • Dense passage retrieval: Retrieves text passages using dense vector similarity rather than exact keywords.
  • Contrastive retriever training: Pulls matching query-document pairs together and pushes nonmatches apart.
  • Hard-negative mining: Trains on confusing wrong results so the retriever learns finer distinctions.
  • Reranking: Reorders an initial candidate set with a stronger but slower model.
  • Hybrid sparse-dense retrieval: Combines lexical matching with semantic embedding search.
  • Reciprocal-rank fusion: Merges multiple ranked lists by rewarding documents that rank well across systems.
  • Metadata filtering: Restricts search by fields such as date, source, permission, topic, or user segment.
  • Multivector retrieval: Stores multiple vectors per document to capture different passages, fields, or token-level signals.

Approximate nearest-neighbour algorithms

  • Locality-sensitive hashing: Hashes similar vectors into the same buckets with high probability.
  • Product quantization: Compresses vectors into compact codes for memory-efficient similarity search.
  • Inverted file indexes: Partition vector space into clusters and search only promising partitions.
  • Hierarchical navigable small worlds: Build graph indexes that navigate quickly toward nearby vectors.
  • ScaNN-style partitioning and scoring: Uses partitioning, quantization, and optimized scoring for fast large-scale retrieval.

8.3 RAG architecture

Understand the complete pipeline:

  1. Ingestion: Bring documents, records, pages, or data feeds into the retrieval system.
  2. Parsing: Extract usable text, tables, structure, and attachments from raw files.
  3. Chunking: Split content into retrieval-sized units that preserve enough context.
  4. Metadata extraction: Attach source, date, permissions, topic, author, and other filters to each chunk.
  5. Embedding: Convert chunks into vectors for semantic search.
  6. Indexing: Store sparse, dense, or hybrid searchable representations.
  7. Query rewriting: Reformulate the user request into better retrieval queries.
  8. Retrieval: Pull candidate chunks that may answer the request.
  9. Reranking: Sort candidates by a stronger relevance model before generation.
  10. Context packing: Fit the best evidence into the model context without wasting tokens.
  11. Generation: Ask the model to answer using the supplied evidence.
  12. Citation or provenance generation: Link claims back to source chunks or documents.
  13. Faithfulness evaluation: Check whether the answer is supported by retrieved evidence.
  14. Cache and index refresh: Keep results fast and update indexes when source content changes.

Study

  • Naive RAG: Retrieves a few chunks and passes them directly to a generator.
  • Hybrid RAG: Combines lexical, dense, metadata, and reranking signals.
  • Multi-query retrieval: Runs several query variants to improve recall.
  • Query decomposition: Breaks complex questions into smaller retrieval subquestions.
  • Parent-child retrieval: Retrieves small chunks but returns larger parent context for generation.
  • Graph RAG: Uses entities and relationships to retrieve or summarize connected knowledge.
  • Agentic retrieval: Lets an agent decide when and how to search while solving a task.
  • Iterative retrieval: Searches, reasons, and searches again as new information is needed.
  • Corrective RAG: Detects weak retrieval and repairs it through rewriting, fallback search, or validation.
  • Retrieval routing: Sends queries to the right index, tool, retriever, or knowledge source.
  • Context compression: Shortens retrieved evidence while preserving the pieces needed for the answer.
  • Lost-in-the-middle effects: Long contexts can cause models to ignore evidence placed away from the beginning or end.

Learning target

You should be able to compare lexical, dense, hybrid, and reranking retrieval systems, explain approximate nearest-neighbour tradeoffs, and design a RAG pipeline that preserves provenance and evaluates faithfulness.
