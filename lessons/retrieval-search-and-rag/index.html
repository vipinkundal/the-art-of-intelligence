Phase 8

8. Retrieval, Search And RAG

Retrieval-augmented generation was proposed as a way to combine a parametric language model with non-parametric external memory.

8.1 Classical information retrieval

Study:

  • Inverted indexes
  • Term frequency
  • Inverse document frequency
  • TF-IDF
  • BM25
  • Boolean retrieval
  • Tokenization and stemming
  • Fielded search
  • Query expansion
  • Learning to rank

TF-IDF

tfidf(t, d) = tf(t, d) log(N / df(t))

BM25

score(D, Q) = sum_{q in Q} IDF(q) * f(q, D)(k_1 + 1) / (f(q, D) + k_1(1 - b + b|D| / avgdl))

Learn:

  • Bi-encoders

Approximate nearest-neighbour algorithms

  • Locality-sensitive hashing
  • Product quantization

  • Hierarchical navigable small worlds
  • ScaNN-style partitioning and scoring

8.3 RAG architecture

Understand the complete pipeline:

  1. Ingestion
  2. Parsing
  3. Chunking
  4. Metadata extraction
  5. Embedding
  6. Indexing
  7. Query rewriting
  8. Retrieval
  9. Reranking
  10. Context packing
  11. Generation
  12. Citation or provenance generation
  13. Faithfulness evaluation
  14. Cache and index refresh

Study

  • Naive RAG
  • Hybrid RAG
  • Multi-query retrieval
  • Query decomposition
  • Parent-child retrieval
  • Graph RAG
  • Agentic retrieval
  • Iterative retrieval
  • Corrective RAG
  • Retrieval routing
  • Context compression
  • Lost-in-the-middle effects

Learning target

You should be able to compare lexical, dense, hybrid, and reranking retrieval systems, explain approximate nearest-neighbour tradeoffs, and design a RAG pipeline that preserves provenance and evaluates faithfulness.
